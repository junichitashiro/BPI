PHASE 01

現状把握

Ph.1 — Excel
ヒアリングシート
訪問前の事前情報整理から、ヒアリング当日の質問・記録、現場観察まで3シートで一体管理。
  • 1_事前情報：会社概要・キーパーソン・事前課題
  • 2_ヒアリング記録：A〜E 5セクションの質問リスト
  • 3_現場観察記録：ジェムバウォーク用チェック項目
Ph.1 — Excel
現状業務フロー図
工程順序・担当・使用設備・標準時間を一覧化し、情報の流れと問題ポイントを記録。フロー図の貼り付けエリアも付属。
  • 1_工程フロー記録：工程一覧・情報の流れ・問題メモ
  • 2_フロー図スケッチ用：手描き・他ツールからの貼り付け
Ph.1 — Excel
工数記録表
工程ごとに最大5回分の作業時間を計測・記録し、平均・最小・最大を自動計算。日別サマリシートで集計も可能。
  • 1_工数記録：工程別 計測値・自動集計
  • 2_日別サマリ：曜日別工数・時間換算
  • 気づき・分析メモ欄付き
Ph.1 — Excel
現状サマリー
ヒアリング・観察・計測の結果を8セクションで統合整理。改善テーマ候補の記載でPhase 2への引き渡し文書としても機能。
  • 会社情報・業務フロー・困り事の統合整理
  • 定量データ（工数・不良率・LT）欄
  • 改善テーマ候補・残課題・総合所見
PHASE 02

課題定義

Ph.2 — Excel
問題構造シート
なぜなぜ分析またはフィッシュボーン図で問題を構造化。症状と根本原因を切り分けて記録する。
  • なぜなぜ分析記録欄（最大5段階）
  • フィッシュボーン図スケッチエリア
  • 症状 vs 根本原因の切り分け整理
Ph.2 — Excel
課題一覧・優先順位表
抽出した課題をインパクト×実現性マトリクスで優先順位づけ。着手順を視覚的に整理する。
  • 課題一覧テーブル（原因・影響・対象工程）
  • インパクト×実現性マトリクス
  • 優先度スコアリング欄
Ph.2 — Excel
改善テーマ定義書
「何を・なぜ・どの範囲で改善するか」を1枚で定義。As-Is / To-Be・KPI・スコープを明文化してPhase 3の起点とする。
  • テーマ名称・対象工程・背景
  • 現状（As-Is）と目標（To-Be）
  • 成功指標・スコープ・制約条件
PHASE 03

改善設計

Ph.3 — Excel
あるべき姿定義書
現状（As-Is）とあるべき姿（To-Be）を項目ごとに対比して記録。数値でKPIを合意するための基準文書。
  • As-Is / To-Be 対比テーブル
  • 成功基準・KPI（現状値・目標値・計測方法）
  • スコープ外・制約条件の明記欄
Ph.3 — Excel
解決策比較表
複数の解決策を評価軸（効果・コスト・期間・負荷など）で比較。◎○△×評価と総合スコアで採用案を選定・記録する。
  • 評価軸×案の比較マトリクス
  • 総合スコアの自動計算
  • 採用根拠・不採用理由の記録欄
Ph.3 — Excel
実施計画書
WBS・ガントチャート・スコープ・担当・リソースを1枚に集約。関係者との合意取りと進捗管理に使う。
  • WBS・ガントチャート（週単位）
  • スコープ・担当者・役割分担
  • 必要リソース・予算・制約条件
PHASE 04

実施・検証

Ph.4 — Excel
パイロット実施記録
パイロット運用中の実施内容・気づき・問題を日次で記録。問題ログと対処状況を一体管理する。
  • 実施ログ（日付・担当・内容・気づき）
  • 問題ログ（内容・影響度・対処・ステータス）
  • 中間評価・計画との乖離確認欄
Ph.4 — Excel
効果測定シート
Before/After比較と改善率の自動計算、目標達成判定（継続 or 再設計）まで一枚で完結。
  • 指標別 Before / After 比較テーブル
  • 改善率・達成率の自動計算
  • 目標達成判定・次アクション記録
PHASE 05

定着・展開

Ph.5 — Excel
標準作業手順書（SOP）
改善後の作業を番号付きステップで標準化。各手順にポイント・使用工具・注意事項を付記し、誰でも再現できる形にする。
  • 作業手順のステップ記述（番号付き）
  • 各ステップの注意点・工具・判断基準
  • 版管理・改訂履歴欄
Ph.5 — Excel
引き渡し確認チェックリスト
改善の引き渡し完了条件をチェックリスト化。SOP整備・OJT実施・モニタリング体制の確立を確認してサインオフする。
  • 引き渡し完了条件のチェック項目
  • OJT実施記録・担当者確認欄
  • 支援者・現場担当者のサインオフ
Ph.5 — Excel
横展開検討メモ
今回の改善を他ライン・他現場へ展開するための候補整理と優先度評価。教訓・申し送りとKPT振り返りを記録する。
  • 横展開候補一覧（適用可否・優先度）
  • 教訓・成功要因・失敗要因の記録
  • KPT振り返り（Keep/Problem/Try）
